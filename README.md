# RISC CPU Design Project

A project for designing and simulating a RISC CPU architecture.

## 1. Directory Structure
- **code/**: Contains the design source code (`src.v`) and the testbench (`tb.v`).
- **testcases/**: Sample programs (`PROG1.txt`, `PROG2.txt`, `PROG3.txt`) to verify CPU operations.
- **outputs/**: Synthesis results and reports (Area, Power, Timing, QoR) generated by Cadence tools.
- **scripts**: `run.tcl` used for running Cadence automation scripts.

## 2. Tools and Workflow

### Functional Simulation
- **Tool**: Xilinx Vivado.
- **Execution**: Use `src.v` and `tb.v` to verify the functional correctness of the logic design.

### Synthesis & Analysis
- **Tool**: Cadence (Genus/Innovus).
- **Execution**: 
    - Execute `testcases` to evaluate performance.
    - Generate reports in the `outputs/` directory, including:
        - `area.rpt`: Area utilization report.
        - `power.rpt`: Power consumption report.
        - `timing.rpt`: Timing analysis report (slack, delay).
        - `risc_cpu_m.v`: Post-synthesis netlist.

## 3. Usage Instructions
1. Use **Vivado** to open the source code and run functional simulation.
2. Use **Cadence** to load `run.tcl` and constraints from `risc_cpu.sdc` to perform synthesis and export reports.
